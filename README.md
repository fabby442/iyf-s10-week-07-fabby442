## Project Description
This Week 7 submission includes multiple JavaScript applications built to master state management, data persistence, modular architecture, and clean code practices.

The projects simulate real-world frontend development patterns without frameworks, focusing on JavaScript fundamentals.

- HTML5
- CSS3
- JavaScript (ES Modules)

- sessionStorage API

- Prettier (code formatting)

# 🧠 Projects Included

---

## 1. To-Do List App (Persistent)

### Features
- Add tasks
- Delete tasks
- Mark tasks as complete
- Filter tasks (if implemented)
- Persistent storage using localStorage

### Lessons Learned
- CRUD operations in JavaScript
- Managing array-based state
- DOM updates based on state changes
- Persisting UI state using localStorage

---

## 2. Shopping Cart App

### Features
- Product listing with images
- Add/remove items
- Increase/decrease quantity
- Automatic total calculation
- Persistent cart using localStorage
- Modular JavaScript structure

### Lessons Learned
- State management in complex UI
- Splitting logic into modules
- Syncing UI with state
- Preventing duplicated logic

---

## 3. Session Storage Form Auto-Save

### Features
- Auto-saves form input while typing
- Restores data on refresh
- Clears data on submit
- Uses sessionStorage (temporary persistence)

### Lessons Learned
- Difference between sessionStorage and localStorage
- Real-time data saving patterns
- Form state management

---

## 4. Code Quality & Tooling Setup

### Features
- ESLint configured for JavaScript linting
- Prettier for consistent formatting
- Clean code practices enforced

### Lessons Learned
- Identifying and fixing code issues automatically
- Enforcing coding standards
- Writing maintainable JavaScript

# 🚀 Challenges Faced

- Keeping UI in sync with state
- Managing persistent storage correctly
- Fixing ESLint errors and enforcing rules
- Structuring multi-file JavaScript architecture
- Handling quantity and cart logic correctly

---

# 🧠 Lessons Learned

- JavaScript applications should be state-driven, not DOM-driven
- localStorage is essential for persistence
- Code structure matters as much as functionality
- Linting tools improve code quality significantly
- Small modular functions make debugging easier
